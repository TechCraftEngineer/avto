# Руководство по разработке

## Структура компонентов

Проект использует централизованную систему экспортов компонентов через `apps/app/src/components/index.ts`. Все компоненты должны импортироваться через этот файл.

### Категории компонентов:

- **UI Components**: Базовые UI компоненты (кнопки, формы, модалы)
- **Layout Components**: Компоненты разметки (хедеры, сайдбары)
- **Chat Components**: Компоненты чата и общения
- **Gig Components**: Компоненты для работы с вакансиями
- **Interview Components**: Компоненты для интервью
- **Recruiter Components**: Компоненты для рекрутеров
- **Vacancy Components**: Компоненты вакансий
- **Settings Components**: Компоненты настроек
- **Response Components**: Компоненты откликов

## Правила импортов

### ✅ Правильно:
```typescript
import { Button, Card, SafeHtml } from "~/components";
```

### ❌ Неправильно:
```typescript
import { Button } from "~/components/ui/components/button";
import { SafeHtml } from "~/components/safe-html";
```

## Сборка и очистка

### Очистка перед сборкой:
```bash
bun run clean:build
```

### Полная очистка (включая turbo cache):
```bash
bun run clean:build --full
```

## Производительность

### Компоненты с ленивой загрузкой:
- Используйте `React.lazy()` для тяжелых компонентов
- Группируйте компоненты по маршрутам

### Оптимизация импортов:
- Все импорты идут через централизованный index.ts
- Избегайте глубоких путей импортов
- Используйте tree-shaking

## Качество кода

## Архитектурные принципы

1. **Единая точка входа**: Все компоненты экспортируются через index.ts
2. **Логическая группировка**: Компоненты группируются по функциональности
3. **Изоляция**: Каждый компонент имеет четкую ответственность
4. **Переиспользование**: Компоненты проектируются для переиспользования

## Добавление новых компонентов

1. Создайте компонент в соответствующей папке
2. Добавьте экспорт в `components/index.ts` в правильную категорию
3. Обновите документацию если необходимо
4. Убедитесь, что все импорты используют централизованный путь

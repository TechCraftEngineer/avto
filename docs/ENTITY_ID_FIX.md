# Исправление проблемы с entity_id в откликах

## Проблема

При импорте архивных вакансий возникала ситуация, когда:
1. Вакансия импортировалась с ID `A`
2. Отклики сохранялись с `entity_id = A`
3. Вакансия удалялась или импортировалась заново с ID `B`
4. Отклики оставались с `entity_id = A`, который больше не существует

Это приводило к тому, что отклики не отображались в интерфейсе, так как их `entity_id` не соответствовал ни одной существующей вакансии.

## Диагностика

Для диагностики проблемы созданы скрипты:

### 1. Диагностика откликов с несуществующими entity_id

```bash
bun run scripts/diagnose-entity-id.ts
```

Скрипт проверяет:
- Отклики с несуществующими `entity_id`
- Статистику по источникам импорта
- Распределение откликов по вакансиям

### 2. Поиск удаленной вакансии

```bash
bun run scripts/find-missing-vacancy.ts
```

Скрипт пытается найти подходящую вакансию по `externalId` для откликов с несуществующим `entity_id`.

## Решение

### Ручное исправление

Для исправления существующих откликов:

```bash
bun run scripts/fix-entity-id.ts
```

Скрипт обновляет `entity_id` со старого значения на новое для всех затронутых откликов.

### Автоматическое предотвращение

В функцию `saveBasicResponse` добавлена логика, которая:
1. Проверяет, существует ли отклик с таким `resumeId` для любой вакансии
2. Если отклик существует, но с другим `entity_id`, обновляет его
3. Логирует изменение в историю откликов

Это предотвращает проблему при повторном импорте вакансий.

## Результат

После исправления:
- Все 345 откликов были обновлены с правильным `entity_id`
- Отклики теперь отображаются в интерфейсе
- Добавлена защита от повторения проблемы в будущем

## Причина проблемы

Проблема возникла из-за того, что:
1. При импорте архивной вакансии создается новая запись с новым UUID
2. Функция `saveBasicVacancy` ищет существующую вакансию по `workspaceId`, `source` и `externalId`
3. Если вакансия была удалена или импортирована в другой workspace, создается новая запись
4. Старые отклики остаются со старым `entity_id`

## Рекомендации

1. **Не удаляйте вакансии с откликами** - используйте архивирование
2. **При повторном импорте** - функция автоматически обновит `entity_id` в откликах
3. **Периодически запускайте диагностику** - для выявления проблем на ранней стадии

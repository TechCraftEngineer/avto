# Исправление проблемы с entity_id в откликах

## Что было сделано

### 1. Диагностика проблемы ✅
- Создан скрипт `scripts/diagnose-entity-id.ts` для выявления откликов с несуществующими entity_id
- Обнаружено 345 откликов с неправильным entity_id
- Все отклики ссылались на удаленную вакансию `019c1073-4afe-7627-8fb4-6d4d521c93b1`

### 2. Поиск правильной вакансии ✅
- Создан скрипт `scripts/find-missing-vacancy.ts`
- Найдена правильная вакансия: "Няня для двух девочек 4 и 6 лет (м. Бутырская)"
- Новый ID: `019c1f17-0a0d-744a-9b92-7304626ca922`
- External ID: `128580152`

### 3. Исправление данных ✅
- Создан скрипт `scripts/fix-entity-id.ts`
- Обновлено 345 откликов с правильным entity_id
- Все отклики теперь корректно связаны с вакансией

### 4. Предотвращение проблемы в будущем ✅
- Улучшена функция `saveBasicResponse` в `packages/jobs/src/services/response/response-repository.ts`
- Добавлена проверка существующих откликов по `resumeId`
- При обнаружении отклика с другим entity_id происходит автоматическое обновление
- Изменения логируются в историю откликов

## Причина проблемы

При повторном импорте архивной вакансии:
1. Создавалась новая запись вакансии с новым UUID
2. Старые отклики оставались со старым entity_id
3. Отклики не отображались в интерфейсе

## Результат

- ✅ Все 345 откликов исправлены
- ✅ Отклики теперь отображаются в интерфейсе
- ✅ Добавлена защита от повторения проблемы
- ✅ Создана документация в `docs/ENTITY_ID_FIX.md`

## Измененные файлы

1. `packages/jobs/src/services/response/response-repository.ts` - улучшена функция `saveBasicResponse`
2. `scripts/diagnose-entity-id.ts` - новый скрипт для диагностики
3. `scripts/find-missing-vacancy.ts` - новый скрипт для поиска вакансии

```bash

# Поиск вакансии для откликов

# Исправление (если нужно)

```

## Обзор страницы

Страница детального отклика на gig находится по адресу:
```
/orgs/[orgSlug]/workspaces/[slug]/gigs/[gigId]/responses/[responseId]
```

### Текущая структура страницы

| Компонент | Описание |
|-----------|----------|
| **Header Card** | Имя кандидата, статус, рейтинг, бейджи (портфолио, цена) |
| **Key Info Summary** | Предложенная цена и сроки в карточках |
| **Tabs** | Анализ, Объяснение, Диалог, Предложение, Опыт, Портфолио, Контакты |
| **Action Buttons** | Принять, Отклонить, Связаться, Отправить приветствие, Оценить |
| **Dialogs** | Подтверждение принятия/отклонения, Отправка сообщения |

### Данные, отображаемые на странице

- Информация о кандидате (имя, ID)
- Статус отклика (NEW, EVALUATED, INTERVIEW, NEGOTIATION, COMPLETED, SKIPPED)
- Предложенная цена и сроки выполнения
- Портфолио (ссылки и файлы)
- Скрининг-оценка (общий балл, анализ по факторам)
- Интервью-оценка (если есть)
- История чата с кандидатом
- Профиль кандидата (распознанный из резюме)

## Сильные стороны текущей реализации

### ✅ Адаптивный дизайн
- Использование responsive классов (`sm:`, `lg:`) для корректного отображения на разных устройствах
- `min-h-[44px]` для мобильных устройств (соответствует рекомендациям по доступности)
- `touch-action-manipulation` для предотвращения двойного зума на мобильных

### ✅ Состояния загрузки и ошибок
- Skeleton-компоненты для всех секций
- Информативные сообщения об ошибках
- Состояние "Отклик не найден" с кнопкой возврата

### ✅ Интерактивность
- Polling-механизм для обновления данных интервью-оценки (каждые 5 секунд, до 1 минуты)
- Optimistic updates для действий accept/reject
- Диалоговые окна для подтверждения критических действий

### ✅ Доступность
- ARIA-атрибуты на интерактивных элементах
- Клавиатурная навигация
- семантический HTML

### ✅ Безопасность
- Защита от XSS в портфолио-ссылках (нормализация URL)
- Валидация на стороне сервера через tRPC

## Проблемные места и рекомендации

### 1. Визуальный дизайн и UI

#### Проблема 1.1: Перегруженность интерфейса
**Описание:** На странице отображается слишком много вкладок (до 7 штук), что может затруднить быстрое принятие решения.

**Рекомендация:**
```tsx
// В файле tabs.tsx
// Реализовать "умную" сортировку вкладок:
// 1. Показывать по умолчанию только 3-4 самые важные вкладки
// 2. Добавить выпадающий список "Ещё" для остальных вкладок
// 3. Запоминать последний выбор пользователя в localStorage

const defaultTabs = ['proposal', 'experience', 'analysis'];
const additionalTabs = ['explanation', 'dialog', 'portfolio', 'contacts'];
```

**Приоритет:** Средний  
**Влияние на UX:** Высокое - упрощает принятие решений

#### Проблема 1.2: Отсутствие контекста Gig
**Описание:** На странице отклика не отображается информация о самом gig (название, бюджет, сроки), к которой кандидат откликался.

**Рекомендация:**
```tsx
// Добавить в header-card.tsx секцию с информацией о gig
<div className="bg-muted/50 p-3 rounded-lg">
  <p className="text-sm font-medium">Отклик на: {gigTitle}</p>
  <p className="text-xs text-muted-foreground">
    Бюджет: {gigBudgetMin} - {gigBudgetMax} ₽ | Срок: {gigTimeline} дней
  </p>
</div>
```

**Приоритет:** Высокий  
**Влияние на UX:** Высокое - пользователь видит полный контекст

#### Проблема 1.3: Неинформативный аватар
**Описание:** Вместо фотографии кандидата отображается только иконка пользователя с градиентом.

**Рекомендация:**
```tsx
// Добавить отображение аватара или инициалов
{response.candidateAvatar ? (
  <img src={response.candidateAvatar} alt="" className="h-12 w-12 rounded-full" />
) : (
  <div className="h-12 w-12 rounded-full bg-gradient...">
    {getInitials(response.candidateName)}
  </div>
)}
```

**Приоритет:** Низкий  
**Влияние на UX:** Среднее - персонализация

**Приоритет:** Высокий  
**Влияние на UX:** Высокое - ускоряет принятие решения

#### Проблема 2.3: Жёстко закодированные данные для сравнения
**Описание:** В `pricing-card.tsx` используются моковые данные для сравнения с рынком:

**Приоритет:** Высокий  
**Влияние на UX:** Высокое - достоверность данных

### 3. Навигация

#### Проблема 3.1: Отсутствие хлебных крошек
**Описание:** Есть только кнопка "Назад к откликам", но нет полной навигационной цепочки.

**Рекомендация:**

```tsx
    Gig-и
  </Link>
  <ChevronRight className="h-4 w-4" />
  <Link href={`/orgs/${orgSlug}/workspaces/${workspaceSlug}/gigs/${gigId}`}>

```

**Приоритет:** Средний  
**Влияние на UX:** Среднее - улучшает ориентацию

#### Проблема 3.2: Нет навигации между откликами
**Описание:** Нельзя быстро переключаться между откликами без возврата к списку.

**Рекомендация:**
```tsx
// Добавить кнопки предыдущий/следующий отклик
<div className="fixed bottom-4 left-1/2 -translate-x-1/2 flex gap-2">
  <Button variant="outline" onClick={navigateToPrevResponse}>
    <ChevronLeft className="h-4 w-4" />
    Предыдущий
  </Button>
  <span className="py-2">{currentIndex} из {total}</span>
  <Button variant="outline" onClick={navigateToNextResponse}>
    Следующий
    <ChevronRight className="h-4 w-4" />
  </Button>
</div>
```

**Приоритет:** Средний  
**Влияние на UX:** Высокое - ускоряет работу с откликами

#### Проблема 3.3: Несохранённое состояние при возврате
**Описание:** При возврате к списку откликов позиция скролла и фильтры не сохраняются.

**Рекомендация:**
```tsx
// Использовать URL-параметры для сохранения состояния
// В responses/page.tsx добавить:
const scrollPosition = searchParams.get('scroll');
useEffect(() => {
  if (scrollPosition) {
    window.scrollTo(0, parseInt(scrollPosition));
  }
}, [scrollPosition]);

// При уходе со страницы:
const handleBack = () => {
  const url = new URL(searchParams);
  url.searchParams.set('scroll', window.scrollY.toString());
  router.push(url.toString());
};
```

**Приоритет:** Низкий  
**Влияние на UX:** Среднее

### 4. Призывы к действию (CTA)

**Описание:** В header расположено 6+ кнопок действий, что создаёт когнитивную перегрузку.

**Рекомендация:**
```tsx
// Сгруппировать действия по приоритету:
// 1. Primary: "Выбрать" - всегда видима, выделена
// 2. Secondary: "Связаться", "Принять", "Отклонить" - в выпадающем меню
// 3. Tertiary: "Оценить", "Kwork чат" - в меню "Ещё"

<div className="flex gap-2">
  <Button variant="default" onClick={onAccept}>Выбрать</Button>
  <DropdownMenu>
    <DropdownMenuTrigger asChild>
      <Button variant="outline">
        <MoreHorizontal className="h-4 w-4" />
      </Button>
    </DropdownMenuTrigger>
    <DropdownMenuContent>
      <DropdownMenuItem onClick={onMessage}>Связаться</DropdownMenuItem>
      <DropdownMenuItem onClick={onReject}>Отклонить</DropdownMenuItem>
      <DropdownMenuItem onClick={onEvaluate}>Оценить</DropdownMenuItem>
    </DropdownMenuContent>
  </DropdownMenu>
</div>
```

**Приоритет:** Высокий  
**Влияние на конверсию:** Высокое - чёткий фокус на главном действии

#### Проблема 4.2: Нет контекстных подсказок
**Описание:** Кнопки "Принять" и "Отклонить" не показывают, почему стоит или не стоит принимать кандидата.

**Рекомендация:**

**Приоритет:** Средний  
**Влияние на конверсию:** Среднее

```tsx
    response_id: responseId,

const handleViewDetails = () => {

```

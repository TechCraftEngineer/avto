# Обоснования для публикации в Google Web Store

Документ содержит тексты для вкладки **«Меры по обеспечению конфиденциальности»** расширения «Recruitment Assistant».

---

## Необходимые обоснования разрешений

### 1. activeTab

**Обоснование:**

Разрешение «activeTab» требуется для временного доступа к активной вкладке при нажатии пользователем на иконку расширения. Оно используется для:

- определения контекста страницы (LinkedIn, HeadHunter, резюме или вакансии) и показа соответствующих элементов интерфейса в popup;
- отправки сообщений content script на активной вкладке для получения данных (например, выбранные вакансии, количество откликов);
- выполнения действий только на той странице, которую пользователь сейчас просматривает.

Доступ предоставляется только после явного действия пользователя (клик по иконке) и действует только в момент открытия popup.

---

### 2. scripting

**Обоснование:**

Разрешение «scripting» используется для программного внедрения content script в вкладки HeadHunter Employer в следующих случаях:

- Пользователь нажимает «Импорт выбранных вакансий» или «Импорт откликов» в popup — скрипт может ещё не быть загружен в вкладку. Расширение явно внедряет скрипт (`hh-employer-content-script.ts`), чтобы выполнить импорт без перезагрузки страницы.
- Скрипт загружается только из файлов самого расширения, указанных в `manifest.json`; внешние или удалённые скрипты не используются.

Использование ограничено действиями пользователя и нужными страницами (hh.ru/employer).

---

### 3. storage

**Обоснование:**

Разрешение «storage» используется для локального хранения данных только на устройстве пользователя:

- **Токен авторизации** (authToken) и данные пользователя (userData) — для входа и работы с API приложения;
- **Идентификатор рабочего пространства** (workspaceId) — настройки пользователя;
- **Выбранные вакансии** — для массового импорта в приложение;
- **Прогресс импорта** — для отображения статуса во время длительных операций.

Данные сохраняются в `chrome.storage.local`, не передаются сторонним серверам и не синхронизируются в облако. Событие `chrome.storage.onChanged` используется для обновления интерфейса popup при изменении данных.

---

### 4. tabs

**Обоснование:**

Разрешение «tabs» используется для:

- **Определения активной вкладки** — `chrome.tabs.query({ active: true, currentWindow: true })` для получения URL и контекста страницы при открытии popup;
- **Отправки сообщений** — `chrome.tabs.sendMessage(tabId, ...)` для общения с content script (импорт вакансий, откликов, получение данных профиля);
- **Открытия новых вкладок** — `chrome.tabs.create` для перехода к авторизации, страницам HeadHunter Employer и LinkedIn по действиям пользователя;
- **Закрытия вкладки** — после OAuth callback страница перенаправления закрывается через `chrome.tabs.remove`;
- **Слушателей событий** — `chrome.tabs.onActivated`, `chrome.tabs.onUpdated` для обновления интерфейса при смене вкладки или URL.

Доступ к вкладкам используется только для работы расширения и не связан со сбором личных данных для сторонних целей.

---

### 5. Host permissions (доступ к хостам)

**Обоснование использования каждого домена:**

| Домен | Назначение |
|-------|------------|
| `https://www.linkedin.com/*`, `https://linkedin.com/*` | Парсинг профилей кандидатов, отображение панели расширения на страницах `/in/*` |
| `https://hh.ru/*`, `https://www.hh.ru/*`, `https://*.hh.ru/*` | Парсинг резюме и откликов, работа с интерфейсом работодателя (hh.ru/employer), вызов API Chatik для получения чатов и поиска |
| `https://*.avtonaim.qbsoft.ru/*` | Связь с основным приложением: авторизация OAuth, API для импорта кандидатов, вакансий и откликов |
| `http://localhost:3000/*` | Поддержка локальной разработки и тестирования приложения |

Все запросы к API ограничены явным whitelist доменов в коде (в т.ч. защита от SSRF). Данные с этих доменов обрабатываются только в рамках функций расширения (импорт, синхронизация с Avtonaim).

---

### 6. Использование «удалённого кода» (remote code)

**Обоснование:**

Расширение использует внедрение скриптов в контекст страницы, но **не выполняет код, загруженный из интернета**:

- Используются только скрипты из пакета расширения: `fetch-page-context.js` и `spoiler-chat-interceptor.js` (через `chrome.runtime.getURL()`).
- Эти файлы включены в `web_accessible_resources` и загружаются локально из расширения.
- Внедрение необходимо для:
  1. Обхода Content Security Policy на hh.ru — для выполнения `fetch` с куками пользователя при загрузке HTML резюме и вакансий;
  2. Перехвата ответов API HeadHunter при клике по «Показать сообщения» (spoiler) — для извлечения сопроводительных писем откликов.

Никакой код не загружается с внешних серверов; не используются `eval()`, `new Function()` или `dangerouslySetInnerHTML`. См. также `docs/SECURITY.md`.

---

### 7. Описание цели расширения

**Краткое описание цели (для поля «Цель»):**

Расширение помогает рекрутерам автоматизировать работу с профилями кандидатов на LinkedIn и HeadHunter: парсинг профилей и резюме, импорт вакансий и откликов в систему Avtonaim, синхронизация сопроводительных писем и чатов. Все операции выполняются по действиям пользователя; данные хранятся локально или передаются только в связанное приложение Avtonaim.

## Прочие замечания Google Web Store

### Подробное описание (минимум 25 символов)

Google указывает: «Подробное описание слишком короткое или отсутствует. Минимально допустимое количество символов – 25.» Скорее всего, имеется в виду либо краткое описание (минимум ~132 символа), либо длинное — несколько предложений. Ниже — расширенный текст для поля «Подробное описание»:

---

**Подробное описание расширения:**

Recruitment Assistant — расширение для автоматизации работы рекрутеров с профилями кандидатов на LinkedIn и HeadHunter.

Основные возможности:
• Парсинг профилей на LinkedIn и резюме на HeadHunter — извлечение имени, контактов, опыта и навыков.
• Импорт вакансий и откликов с HeadHunter Employer в систему Avtonaim.
• Синхронизация сопроводительных писем и чатов с кандидатами.
• Панель на страницах кандидатов для быстрого добавления в Avtonaim.

Расширение работает только по действиям пользователя, хранит данные локально и взаимодействует только с приложением Avtonaim. Подходит для рекрутеров и HR-специалистов, использующих Avtonaim.

- Укажите и подтвердите контактный email в разделе **«Аккаунт»**.
- Убедитесь, что email отображается в карточке расширения (при необходимости — в настройках приватности).

### Подтверждение использования данных

- На вкладке **«Конфиденциальность»** подтвердите соответствие [Правилам программы для разработчиков](https://developer.chrome.com/docs/webstore/program-policies/) в части обработки данных.
- Укажите, собирает ли расширение персональные данные и как они используются (см. текст выше).

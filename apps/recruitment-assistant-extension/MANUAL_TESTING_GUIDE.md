# Руководство по ручному тестированию адаптеров

Это руководство поможет вам протестировать LinkedIn и HeadHunter адаптеры на реальных страницах профилей.

## Подготовка к тестированию

### 1. Сборка расширения

Перед тестированием необходимо собрать расширение:

```bash
cd apps/recruitment-assistant-extension
bun run build
```

После сборки проверьте, что директория `dist/` содержит скомпилированные файлы.

### 2. Загрузка расширения в Chrome

1. Откройте Chrome и перейдите на страницу `chrome://extensions/`
2. Включите "Режим разработчика" (Developer mode) в правом верхнем углу
3. Нажмите "Загрузить распакованное расширение" (Load unpacked)
4. Выберите директорию `apps/recruitment-assistant-extension/dist/`
5. Расширение должно появиться в списке установленных расширений

### 3. Проверка установки

- Убедитесь, что расширение активно (переключатель включен)
- Проверьте, что нет ошибок в консоли расширения
- Иконка расширения должна появиться на панели инструментов Chrome

## Тестирование LinkedIn адаптера

### Тестовые профили

Используйте публичные профили LinkedIn для тестирования. Примеры:
- Профили с полной информацией (имя, должность, опыт, образование, навыки)
- Профили с частичной информацией (некоторые секции отсутствуют)
- Профили с текущей позицией ("по настоящее время")
- Профили с несколькими позициями в одной компании

### Что тестировать

#### 1. Определение страницы профиля

**Тест:** Откройте профиль LinkedIn (URL начинается с `/in/`)

**Ожидаемый результат:**
- Расширение должно определить, что это страница профиля
- Content script должен загрузиться (проверьте в DevTools → Console)

**Как проверить:**
```javascript
// Откройте DevTools (F12) и выполните в консоли:
console.log(window.location.pathname.startsWith('/in/'));
// Должно вернуть true
```

#### 2. Извлечение базовой информации

**Тест:** Проверьте извлечение имени, должности, местоположения и фото

**Ожидаемый результат:**
- Полное имя извлечено корректно
- Текущая должность извлечена корректно
- Местоположение извлечено корректно
- URL фотографии извлечен (если есть)

**Как проверить:**
```javascript
// В консоли DevTools:
const adapter = new LinkedInAdapter();
const basicInfo = adapter.extractBasicInfo();
console.log('Базовая информация:', basicInfo);
```

**Проверьте:**
- `fullName` содержит полное имя
- `currentPosition` содержит текущую должность
- `location` содержит местоположение
- `photoUrl` содержит URL или null

#### 3. Извлечение опыта работы

**Тест:** Проверьте извлечение всех записей об опыте работы

**Ожидаемый результат:**
- Все позиции извлечены
- Для каждой позиции есть: должность, компания, даты, описание
- Текущая позиция имеет `endDate: null`
- Хронологический порядок сохранен

**Как проверить:**
```javascript
const experience = adapter.extractExperience();
console.log('Опыт работы:', experience);
console.log('Количество позиций:', experience.length);
```

**Проверьте:**
- Количество записей соответствует профилю
- Даты корректно распарсены
- Текущая позиция имеет `endDate: null`

#### 4. Извлечение образования

**Тест:** Проверьте извлечение всех записей об образовании

**Ожидаемый результат:**
- Все учебные заведения извлечены
- Для каждого: название, степень, специальность, даты

**Как проверить:**
```javascript
const education = adapter.extractEducation();
console.log('Образование:', education);
```

#### 5. Извлечение навыков

**Тест:** Проверьте извлечение списка навыков

**Ожидаемый результат:**
- Все навыки извлечены
- Порядок сохранен

**Как проверить:**
```javascript
const skills = adapter.extractSkills();
console.log('Навыки:', skills);
console.log('Количество навыков:', skills.length);
```

#### 6. Извлечение контактов

**Тест:** Проверьте извлечение контактной информации

**Ожидаемый результат:**
- Email извлечен (если доступен)
- Телефон извлечен (если доступен)
- Социальные ссылки извлечены (если доступны)

**Примечание:** Контактная информация часто скрыта на LinkedIn и доступна только для подключений.

**Как проверить:**
```javascript
const contacts = adapter.extractContacts();
console.log('Контакты:', contacts);
```

#### 7. Полное извлечение данных

**Тест:** Проверьте извлечение всех данных сразу

**Как проверить:**
```javascript
const allData = adapter.extractAll();
console.log('Все данные:', allData);
console.log('JSON:', JSON.stringify(allData, null, 2));
```

**Проверьте:**
- Все секции заполнены
- `platform` = "LinkedIn"
- `profileUrl` содержит текущий URL
- `extractedAt` содержит дату извлечения

## Граничные случаи для тестирования

### 1. Пустые секции

**Тест:** Найдите профиль без некоторых секций (например, без навыков или образования)

**Ожидаемый результат:**
- Адаптер должен вернуть пустой массив `[]` для отсутствующих секций
- Не должно быть ошибок в консоли

### 2. Текущая позиция

**Тест:** Профиль с текущей работой ("по настоящее время")

**Ожидаемый результат:**
- `endDate` должен быть `null` для текущей позиции
- Парсинг даты должен корректно определить "настоящее время"

### 3. Множественные позиции

**Тест:** Профиль с несколькими местами работы

**Ожидаемый результат:**
- Все позиции извлечены
- Порядок сохранен (обычно от новых к старым)

### 4. Специальные символы

**Тест:** Профиль с специальными символами в именах, должностях

**Ожидаемый результат:**
- Текст извлечен корректно
- Специальные символы не искажены

### 5. Длинные описания

**Тест:** Профиль с длинными описаниями опыта работы

**Ожидаемый результат:**
- Полный текст извлечен
- Нет обрезания текста

## Проверка производительности

### Время извлечения

**Тест:** Измерьте время извлечения данных

**Как проверить:**
```javascript
console.time('extraction');
const data = adapter.extractAll();
console.timeEnd('extraction');
```

**Ожидаемый результат:**
- Время извлечения < 3 секунд (согласно требованию 12.1)

### Использование памяти

**Тест:** Проверьте использование памяти

**Как проверить:**
1. Откройте Chrome Task Manager (Shift + Esc)
2. Найдите процесс расширения
3. Проверьте использование памяти

**Ожидаемый результат:**
- Использование памяти < 50 МБ (согласно требованию 12.4)

## Известные ограничения

1. **Контактная информация:** На обеих платформах контакты часто скрыты и доступны только для подключений/работодателей
2. **Изменения DOM:** Платформы могут изменить структуру страниц, что потребует обновления селекторов
3. **Динамический контент:** Некоторые секции могут загружаться динамически при прокрутке

## Устранение неполадок

### Расширение не загружается

1. Проверьте, что сборка прошла успешно
2. Проверьте консоль расширения на наличие ошибок
3. Перезагрузите расширение на странице `chrome://extensions/`

### Данные не извлекаются

1. Откройте DevTools и проверьте консоль на наличие ошибок
2. Проверьте, что селекторы соответствуют текущей структуре страницы
3. Убедитесь, что страница полностью загружена

### Неполные данные

1. Проверьте, что все секции профиля развернуты
2. Некоторые данные могут быть скрыты настройками приватности
3. Прокрутите страницу вниз, чтобы загрузить весь контент

## Следующие шаги

После успешного ручного тестирования:

1. Задокументируйте найденные проблемы
2. Создайте issue для каждой проблемы
3. Обновите селекторы, если структура страниц изменилась
4. Переходите к следующей фазе разработки (Фаза 4: Извлечение и валидация данных)

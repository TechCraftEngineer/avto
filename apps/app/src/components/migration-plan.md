# План миграции компонентов на новую структуру по доменам

## Текущая структура (проблемы)
- 426 файлов в одном каталоге components/
- Глубокая вложенность (vacancy/responses/use-refresh-subscription.ts)
- Смешанные ответственности в одном домене
- Баррельные импорты (плохо для производительности)

## Новая структура по доменам

```
src/components/
├── auth/                    # Аутентификация
│   ├── components/
│   ├── hooks/
│   └── types/
├── dashboard/               # Дашборд
├── workspace/               # Рабочее пространство
├── organization/            # Организация
├── vacancies/               # Вакансии
│   ├── components/
│   │   ├── vacancy-card/
│   │   ├── vacancy-form/
│   │   └── vacancy-list/
│   ├── hooks/
│   ├── utils/
│   └── types/
├── gigs/                    # Фриланс проекты
├── candidates/              # Кандидаты
├── responses/               # Отклики на вакансии
├── chat/                    # Чат система
├── settings/                # Настройки
├── ui/                      # Переиспользуемые компоненты
│   ├── atoms/              # Button, Input, Badge
│   ├── molecules/          # FormField, Card, Modal
│   └── organisms/          # Header, Sidebar
└── layout/                  # Макет приложения
```

## Приоритеты миграции (по доменам)

### Фаза 1: Базовая структура (1-2 дня)
1. **ui/** - переиспользуемые компоненты
2. **layout/** - компоненты макета
3. **auth/** - аутентификация

### Фаза 2: Основные домены (3-4 дня)
4. **dashboard/** - дашборд
5. **workspace/** - рабочее пространство
6. **organization/** - организация

### Фаза 3: Бизнес-логика (5-7 дней)
7. **vacancies/** - вакансии
8. **gigs/** - фриланс проекты
9. **candidates/** - кандидаты

### Фаза 4: Интеграционные компоненты (3-4 дня)
10. **responses/** - отклики
11. **chat/** - чат система
12. **settings/** - настройки

## Правила миграции

### 1. Прямые импорты (Vercel Best Practice)
```typescript
// ❌ Баррельные импорты
import { Button, Input } from '@/components/ui'

// ✅ Прямые импорты
import Button from '@/components/ui/button'
import Input from '@/components/ui/input'
```

### 2. Структура компонента
```
component-name/
├── component-name.tsx       # Основной компонент
├── component-name.types.ts  # Типы (опционально)
├── component-name.test.tsx  # Тесты (опционально)
├── index.ts                 # Экспорты
└── README.md               # Документация (опционально)
```

### 3. Hooks и утилиты
```
domain/
├── components/             # Компоненты
├── hooks/                  # Кастомные хуки
├── utils/                  # Утилиты
└── types/                  # Общие типы
```

### 4. Server Components по умолчанию
- Все компоненты по умолчанию Server Components
- Client Components только при необходимости
- 'use client' директива только для интерактивных компонентов

## Инструменты миграции

### 1. Скрипт анализа зависимостей
```bash
# Найти все импорты компонентов
find src -name "*.tsx" -o -name "*.ts" | xargs grep -l "from '@/components"
```

### 2. Шаблоны компонентов
- Генерация структуры компонента
- Базовые шаблоны для разных типов компонентов

### 3. Скрипт миграции
- Автоматическое перемещение файлов
- Обновление импортов
- Создание индексных файлов

## Порядок действий для каждого домена

1. **Создать структуру каталогов**
2. **Переместить компоненты**
3. **Обновить импорты**
4. **Создать индексные файлы**
5. **Протестировать**
6. **Удалить старые файлы**

## Критерии успеха

- ✅ Все компоненты перемещены
- ✅ Импорты обновлены
- ✅ Тесты проходят
- ✅ Нет циклических зависимостей
- ✅ Bundle size не увеличился
- ✅ Производительность не ухудшилась

# Components Architecture

Компонентная архитектура приложения, организованная по доменному принципу для обеспечения масштабируемости и поддерживаемости.

## Структура

```
components/
├── ui/                    # Базовые UI компоненты
├── shared/               # Переиспользуемые компоненты
├── layout/               # Компоненты макета
├── auth/                 # Компоненты авторизации
├── candidates/           # Компоненты кандидатов
├── vacancies/            # Компоненты вакансий
├── vacancy/              # Детальные компоненты вакансий
├── gigs/                 # Компоненты gigs
├── chat/                 # Компоненты чата
├── dashboard/            # Компоненты дашборда
├── settings/             # Компоненты настроек
├── funnel/               # Компоненты воронки найма
├── organization/         # Компоненты организации
├── recruiter/            # Компоненты рекрутера
└── index.ts              # Централизованные экспорты
```

## Архитектурные принципы

### 1. Доменная организация
- Компоненты группируются по бизнес-доменам
- Каждый домен имеет собственную папку с полной структурой
- Четкое разделение ответственности

### 2. Стандартизированная структура домена
```
domain/
├── components/           # React компоненты
├── hooks/               # Кастомные хуки
├── utils/               # Утилитарные функции
├── types/               # TypeScript типы
├── README.md            # Документация домена
└── index.ts             # Экспорты домена
```

### 3. Прямые импорты
- Избегайте barrel imports для оптимизации bundle size
- Используйте прямые пути к компонентам
- Централизованные экспорты только в index.ts

### 4. Server Components по умолчанию
- Используйте Server Components для лучшей производительности
- Client Components только при необходимости (интерактивность, хуки)

### 5. Строгая типизация
- TypeScript для всех компонентов
- Брендированные типы для доменных сущностей
- Правильное использование дженериков

## Специальные домены

### UI Components (`ui/`)
Базовые переиспользуемые UI компоненты:
- Диалоги и модальные окна
- Формы и инпуты
- Индикаторы загрузки и состояния

### Shared Components (`shared/`)
Общие компоненты для нескольких доменов:
- Карточки кандидатов
- Детали откликов
- Общие утилиты

### Layout Components (`layout/`)
Компоненты макета приложения:
- Боковая панель
- Шапка сайта
- Навигация

## Производительность

### Bundle Optimization
- Прямые импорты вместо barrel files
- Динамические импорты для тяжелых компонентов
- Оптимизация tree-shaking

### React Best Practices
- Правильное использование Server/Client Components
- Оптимизация re-renders
- Мемоизация тяжелых вычислений

## Использование

```typescript
// Прямой импорт компонента
import { CandidateCard } from '@/components/candidates';

// Импорт из централизованного экспорта
import { CandidateCard } from '@/components';
```

## Добавление новых компонентов

1. Определите домен компонента
2. Создайте папку в соответствующем домене
3. Следуйте стандартизированной структуре
4. Добавьте экспорты в index.ts домена
5. Обновите централизованные экспорты при необходимости

# Реализация обработки ошибок для черновиков вакансий

## Обзор

Реализована централизованная система обработки ошибок для черновиков вакансий с graceful degradation при недоступности сервера.

## Реализованные компоненты

### 1. Серверная часть

#### DraftService.isValidDraft()
**Файл:** `packages/api/src/services/draft.service.ts`

Добавлен метод для валидации структуры черновика:
- Проверяет наличие обязательных полей
- Валидирует структуру draftData
- Проверяет корректность сообщений в истории диалога
- Возвращает false для поврежденных черновиков

**Требования:** 7.3 (Обработка поврежденных черновиков)

#### Интеграция с ErrorHandler
DraftService использует существующий ErrorHandler для:
- Логирования ошибок с контекстом
- Обработки ошибок БД
- Обработки ошибок валидации
- Показа понятных сообщений на русском языке

**Требования:** 1.5, 7.1, 7.2, 7.4

### 2. Клиентская часть

#### DraftErrorHandler
**Файл:** `apps/app/src/utils/draft-error-handler.ts`

Утилита для обработки ошибок на клиенте:

**Типы ошибок:**
- `NETWORK` - Сетевые ошибки (проблемы с подключением)
- `VALIDATION` - Ошибки валидации данных
- `AUTHORIZATION` - Ошибки доступа (401, 403)
- `DATABASE` - Ошибки базы данных
- `CORRUPTED` - Поврежденные черновики
- `UNKNOWN` - Неизвестные ошибки

**Функциональность:**
- Определение типа ошибки
- Маппинг tRPC ошибок на понятные сообщения
- Определение возможности повторной попытки
- Предложение действий (retry, start_new, redirect_to_login)

**Сообщения об ошибках (на русском языке):**
- "Не удалось сохранить изменения. Проверьте подключение к интернету"
- "Не удалось загрузить черновик. Начните создание заново"
- "Временные проблемы с сервером. Попробуйте повторить позже"
- "Произошла ошибка при сохранении данных"
- "Доступ запрещен"

**Требования:** 7.1, 7.2, 7.3, 7.4, 9.4

#### LocalDraftStorage
**Файл:** `apps/app/src/utils/draft-error-handler.ts`

Хранилище для локального резервного копирования (graceful degradation):

**Функциональность:**
- Сохранение черновика в localStorage при сетевых ошибках
- Загрузка черновика из localStorage
- Автоматическое удаление устаревших резервных копий (старше 24 часов)
- Проверка принадлежности черновика пользователю

**Требования:** 7.1, 7.2 (Graceful degradation)

#### Обновленный useDraftPersistence
**Файл:** `apps/app/src/hooks/use-draft-persistence.ts`

Расширенная функциональность хука:

**Новые возможности:**
- Обработка всех типов ошибок через DraftErrorHandler
- Автоматическое сохранение в localStorage при сетевых ошибках
- Автоматическая синхронизация при восстановлении соединения
- Отслеживание состояния ошибок (errorInfo)
- Метод retrySave() для повторной попытки
- Метод syncLocalData() для ручной синхронизации
- Флаг useLocalStorage для индикации локального режима

**Новые возвращаемые значения:**
- `errorInfo` - Информация о текущей ошибке
- `useLocalStorage` - Флаг использования локального хранилища
- `retrySave()` - Повторить попытку сохранения
- `syncLocalData()` - Синхронизировать локальные данные

**Требования:** 1.5, 7.1, 7.2, 7.3, 7.4

#### DraftErrorNotification
**Файл:** `apps/app/src/components/draft-error-notification.tsx`

Компонент для отображения ошибок:

**Функциональность:**
- Визуальное отображение ошибок с иконками и цветами
- Адаптивные действия в зависимости от типа ошибки
- Кнопки "Повторить попытку", "Начать заново", "Закрыть"
- Понятные сообщения на русском языке без англицизмов

**Цветовая схема:**
- Желтый - Сетевые ошибки (предупреждение)
- Оранжевый - Поврежденные черновики
- Красный - Критические ошибки (БД, валидация, доступ)

**Требования:** 7.1, 7.2, 7.3, 9.4

#### LocalStorageWarning
**Файл:** `apps/app/src/components/draft-error-notification.tsx`

Компонент для уведомления о локальном режиме:

**Функциональность:**
- Информирование пользователя о локальном сохранении
- Кнопка "Синхронизировать сейчас" для ручной синхронизации
- Автоматическое скрытие после успешной синхронизации

**Требования:** 7.1, 7.2, 9.4

## Сценарии обработки ошибок

### 1. Сетевая ошибка
1. Попытка сохранения черновика
2. Обнаружение сетевой ошибки
3. Сохранение в localStorage
4. Показ уведомления с кнопкой "Повторить попытку"
5. Автоматическая синхронизация при восстановлении соединения

### 2. Поврежденный черновик
1. Попытка загрузки черновика
2. Валидация структуры (isValidDraft)
3. Обнаружение повреждения
4. Логирование ошибки
5. Удаление поврежденного черновика
6. Показ сообщения "Начните создание заново"

### 3. Ошибка базы данных
1. Попытка сохранения/обновления
2. Ошибка БД (недоступность, таймаут)
3. Retry логика (3 попытки с экспоненциальной задержкой)
4. Логирование с контекстом
5. Показ сообщения "Временные проблемы с сервером"
6. Сохранение в localStorage как резервная копия

### 4. Ошибка авторизации
1. Попытка доступа к черновику
2. Проверка аутентификации
3. Ошибка 401/403
4. Показ сообщения "Доступ запрещен"
5. Опциональный редирект на страницу входа

## Graceful Degradation

При недоступности сервера система:
1. Показывает предупреждение о проблемах с сохранением
2. Сохраняет данные в localStorage как резервную копию
3. Позволяет продолжить работу с AI-ботом
4. Автоматически синхронизирует данные при восстановлении соединения
5. Удаляет локальную копию после успешной синхронизации

## Логирование

Все ошибки логируются с контекстом:
- Тип ошибки
- Сообщение для пользователя
- Технические детали
- userId (если доступен)
- Стек вызовов (для критических ошибок)

# План реализации: Интеграция с ЮКасса

## Обзор

Данный план описывает пошаговую реализацию интеграции с платежной системой ЮКасса. Реализация включает создание схемы базы данных, Zod схем валидации, сервиса для работы с API ЮКасса, tRPC роутера и тестов.

**Текущий статус:** Базовая инфраструктура (схема БД, валидация, клиент API) реализована. Требуется завершить tRPC роутеры и тестирование.

## Архитектура

### Разделение ответственности

**tRPC API** (`packages/api/src/routers/payment/`):
- Создание платежей (create)
- Получение информации о платежах (get, list)
- Проверка статуса платежей (checkStatus)

**Webhook-сервис на Hono** (`apps/webhooks/`):
- Обработка webhook-уведомлений от ЮКасса
- API-верификация webhook
- Обновление статусов платежей в БД
- Независимое масштабирование и деплой

### Преимущества архитектуры

1. **Производительность**: Hono в 3-5 раз быстрее для простых HTTP endpoints
2. **Масштабирование**: Webhook-сервис масштабируется независимо от основного API
3. **Безопасность**: Легче настроить rate limiting и IP-whitelisting
4. **Изоляция**: Проблемы с webhook не влияют на основной API
5. **Гибкость**: Webhook-сервис можно развернуть на edge (Cloudflare Workers, Vercel Edge)

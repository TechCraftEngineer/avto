# Coding Standards

* Использовать только Zod v4
* Не нужно создавать миграции для базы данных
* Не используй npm, только bun
* Не создавай документацию, если тебя не просят
* Запрещаю делать реэкспорты из других пакетов
* **Никакого англицизма в пользовательских текстах**
  * **Область применения**: UI-тексты, сообщения об ошибках, уведомления, документация для конечных пользователей
  * **Исключения**:
    * Названия библиотек и инструментов: Zod, npm, bun, React, TypeScript
    * Технические термины в коде: interface, component, hook, schema, props, state
    * Идентификаторы переменных, функций, API endpoints
    * Устоявшиеся термины без адекватного перевода: email, login, dashboard
    * Список в разделе "Исключения" является приоритетным и применяется независимо от правила о русскоязычных терминах. Если термин попадает в исключения, используйте его как есть, даже если существует устойчивая русская альтернатива.
  * **Примеры**:
    * ✅ Правильно: "Войти в систему", "Электронная почта", "Загрузка данных..."
    * ❌ Неправильно: "Залогиниться", "Имейл", "Лоадинг..."
    * ✅ Правильно в коде: `interface UserProfile`, `const handleLogin = () => {}`
    * ✅ Правильно в документации: "Используйте библиотеку Zod для валидации"
  * **Правило**: Приоритет — консистентность и UX. Если термин понятен пользователю на русском, используйте русский
* Интерфейсы все должны быть понятны рекрутерам
